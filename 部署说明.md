## 🚀 快速部署

### 1. 环境准备

确保阿里云服务器已安装：
- JDK 8 或更高版本
- MySQL 数据库

```bash
# 检查Java版本
java -version

# 检查MySQL服务
systemctl status mysql
```

### 2. 数据库准备

连接到MySQL数据库并创建数据库：

```sql
-- 连接MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE punch CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- 使用数据库
USE punch;

-- 导入数据库结构和初始数据（如果有SQL文件）
-- source /path/to/punch.sql;
```

### 3. 上传文件

将以下文件上传到阿里云服务器：
- `punch.jar` - 应用程序JAR包
- `start.sh` - 启动脚本

```bash
# 创建应用目录
mkdir -p /opt/punch
cd /opt/punch

# 上传文件（使用scp或其他方式）
# scp punch.jar root@39.99.252.67:/opt/punch/
# scp start.sh root@39.99.252.67:/opt/punch/

# 给启动脚本执行权限
chmod +x start.sh
```

### 4. 启动应用

```bash
# 启动应用
./start.sh start

# 查看状态
./start.sh status

# 查看日志
./start.sh logs
```

## 🛡️ 安全配置

### 防火墙设置

确保阿里云安全组开放8080端口：

```bash
# 检查端口是否开放
netstat -tlnp | grep 8080

# 如果使用iptables
iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
```

### HTTPS配置（推荐）

应用已支持HTTPS反向代理，配置步骤：

#### 1. 安装Nginx
```bash
# CentOS/RHEL
yum install -y nginx

# Ubuntu/Debian
apt-get install -y nginx
```

#### 2. 配置Nginx
```bash
# 复制配置文件
cp nginx-punch.conf /etc/nginx/sites-available/punch

# 创建软链接
ln -s /etc/nginx/sites-available/punch /etc/nginx/sites-enabled/

# 修改SSL证书路径（在nginx-punch.conf中）
# ssl_certificate /path/to/your/certificate.crt;
# ssl_certificate_key /path/to/your/private.key;

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
systemctl enable nginx
```

#### 3. 防火墙配置
```bash
# 开放HTTPS端口
firewall-cmd --permanent --add-service=https
firewall-cmd --permanent --add-service=http
firewall-cmd --reload

# 或者直接开放端口
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload
```

## ❗ 故障排除

### 常见问题

1. **端口被占用**
   ```bash
   # 查看端口占用
   netstat -tlnp | grep 8080
   # 杀死占用进程
   kill -9 <PID>
   ```

2. **数据库连接失败**
   - 检查MySQL服务状态
   - 验证数据库用户权限
   - 确认防火墙设置

3. **内存不足**
   ```bash
   # 调整JVM参数（在start.sh中）
   JVM_OPTS="-Xms256m -Xmx512m ..."
   ```

4. **JSP页面无法访问**
   - 确认JAR包包含JSP支持
   - 检查静态资源路径

### 日志分析

```bash
# 查看错误日志
grep -i error punch.log

# 查看启动日志
head -50 punch.log

# 实时监控日志
tail -f punch.log
```
